Featured work

private-ai-infra — an AI authority plane

A local-first AI governance plane that enforces the thesis my day job keeps running into: AI capability is not AI authority. Built end to end — gateway, policy engine, agent control plane, A2A & MCP governance, adversarial evals, and CI. In surveyed enterprise agent deployments, ~99% of configurations contain zero deny rules; this project is the opposite — deny is the default, and every allow is written down.

Enforced in code, proven by attack

A pip-installable, OpenAI-compatible gateway mediates every call to a local model: policy-as-code identity, per-principal model allowlists, an enforced L0–L6 autonomy ceiling, egress guardrails, and a decision audit independently re-verified by a read-only assurance agent. The same plane governs governed agent-to-agent delegation with attenuating authority — peers discover each other from policy-derived A2A Agent Cards and hand off work that can only narrow, never amplify — MCP tool access (per-tool minimum autonomy levels), an evasion-aware ingress prompt-injection firewall, and AI-stack CVE scanning. A 23-case adversarial suite — mapped to the OWASP Top 10 for Agentic Applications and MITRE ATLAS — re-attacks every control and fails CI on regression; 350 tests at 92% coverage back it.
